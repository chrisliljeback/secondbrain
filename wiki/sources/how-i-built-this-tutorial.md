# Source: "Hur jag byggde detta från början" (tutorial transcript)

A YouTube-style walkthrough of building a second brain using Karpathy's method
([[llm-wiki-gist]] / [[llm-wiki-pattern]]) — beginner-friendly, ~7 min.

## Key takeaways
- **The pain it solves:** you save things constantly (articles, videos, midnight
  notes) across five apps and never find them again. The problem isn't note-taking —
  it's that nothing keeps them connected. An LLM does the connecting.
- **Three moves: collect → compile → ask.**
  - *Collect* — gather good sources into one folder.
  - *Compile* — the LLM reads it all and writes a connected wiki.
  - *Ask* — question it; it answers from your own knowledge, with links back to source.
- **Two tools + one folder:** Obsidian (free, to read the vault) + Obsidian Web
  Clipper (browser extension, one-click save from web). The LLM (Claude) is the paid
  piece.
- **YouTube trick:** open a video's transcript, copy it, paste into a text file in
  the folder — now the video's knowledge is in the brain too.
- **Don't overthink volume:** start with ~10 solid sources, not 100; add more later.
- **The setup move:** hand the method to the agent and say *"Read this and set it up
  for me"* (exactly how this vault was created).
- **Best queries to run:** "where do my sources agree and disagree?" and "where are
  the gaps / what should I learn next?" — the brain shows you what's *missing*, not
  just what you know.
- **It's yours:** plain-text markdown files on your computer; no app can take it away.
